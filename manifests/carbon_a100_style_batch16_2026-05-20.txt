Notes
- The benchmark harness is experiments/carbon_a100_style_benchmark.py.
- These runs are closer to the A100 reference than the original batch-1 prompt suite, but still not a controlled accelerator shootout: A100 uses vLLM dynamic batching while Trainium2 uses NxDI static batch 16.
- The most promising optimization follow-up is profile-driven work around the 500M/8B GQA layout warning, attention CTE/TKG eligibility, QKV/output projection, RoPE, and RMSNorm paths. NxDI already selects Neuron attention kernels in this flow, so custom kernels should be introduced only after profiling identifies a real bottleneck.
